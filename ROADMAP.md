# ROADMAP.md

## Roadmap Philosophy

Build the concept before building complexity.

The first product should make the Aspire Rate, Portfolio Rate, and Aspire Gap instantly understandable. Every later feature should deepen trust, specificity, or repeat usage.

## Phase 0: Knowledge Foundation

Status: in progress.

Goals:

- Establish project knowledge system
- Define product language
- Define compliance boundaries
- Lock initial positioning
- Preserve brand standards

Deliverables:

- Root knowledge files
- Agent instructions
- Prompt library
- Research backlog

## Phase 1: Concept Prototype

Goal: prove that users understand and care about their personal inflation rate.

Features:

- Single-page Aspire Rate calculator
- Goal categories with editable assumptions
- Portfolio Rate input
- Aspire Gap output
- Scenario range
- Assumption table
- Shareable summary graphic

Success criteria:

- Users can explain Aspire Rate after one session.
- Users can identify their largest personal inflation driver.
- Users understand that the tool is educational and assumption-based.

## Phase 2: Data-Backed Calculator

Goal: improve trust through sourced defaults.

Features:

- CPI comparison
- Housing default assumptions by geography
- Education cost defaults
- Healthcare cost defaults
- Source citations and timestamps
- User override controls
- Exportable report

Success criteria:

- Users trust the default assumptions enough to edit them.
- Every default has a source and timestamp.
- The product avoids hidden black-box math.

## Phase 3: Scenario Engine

Goal: help users compare possible futures.

Features:

- Save scenarios
- Compare timelines
- Add multiple goals
- Sensitivity analysis
- Best/base/stress case ranges
- Household planning mode

Success criteria:

- Users create more than one scenario.
- Users return to revise assumptions.
- Users share scenarios with partners or advisors.

## Phase 4: Content and Growth Loops

Goal: make Aspire Rate a recognizable financial lens.

Features:

- Public mini-calculators
- Newsletter
- Social chart templates
- City or goal-specific reports
- Editorial explainers
- Embeddable widgets

Success criteria:

- The phrase "your future has its own inflation rate" becomes repeatable.
- Content drives calculator usage.
- Calculator outputs drive content sharing.

## Phase 5: Monetization

Goal: package deeper planning value without violating product boundaries.

Possible features:

- Premium saved scenarios
- Advanced reports
- Data packs
- Advisor collaboration exports
- Team or advisor licenses
- API or embeddable calculator

Success criteria:

- Revenue does not require making investment recommendations.
- Paid features deepen measurement, not advice.
